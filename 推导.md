Leo 5P：5F Startup，5F Active，6F Recovery，-1
Active 任意一帧都能取消成 5P
GGST 攻击输入有 5F Buffer（当前帧+额外4F）
最速 5P→5P

假设第一发 5P 的第一个有效帧（F5）被防住。

因为：

F5 就已经进入可取消状态
你提前按好的 P 会被 Buffer 保存

于是系统会在最早允许取消的那一刻直接开始第二个 5P。

时间轴：

F1~F4 Startup
F5 Active(防住)
     ↓取消
F6 第二个5P Startup1
F7 Startup2
F8 Startup3
F9 Startup4
F10 Active

对手被防住后：

F5 命中Block
F6~F14 Blockstun (9F)

第二个 5P：

F10 就已经打到了

而对手：

F15 才恢复

所以：

F10 打到
F15 对手恢复

中间差了：

15−10=5F

即：

最速 5P→5P 是一个 5F 真连防。

现在讨论放帧

你想做的是：

不让它连防，而是留 Gap。

由于最速时领先 Blockstun 5F：

最速 Gap = -5F

（负数表示还在硬直里就被下一发压到了）

假设你额外等待 d 帧再取消。

那么：

Gap=d−5
验证
延迟0F
0−5=−5

连防。

延迟5F
5−5=0

刚好不连防。

延迟6F
6−5=1

1F Gap。

延迟7F
7−5=2

2F Gap。

延迟8F
8−5=3

3F Gap。

延迟9F
9−5=4

4F Gap。

延迟10F
10−5=5

5F Gap。

所以你真正要延迟多少？

如果目标是社区常说的：

做 1~5F Frame Trap

那么：

想要Gap	需要额外延迟
1F	6F
2F	7F
3F	8F
4F	9F
5F	10F

公式：

延迟=Gap+5
	​

Buffer 对实战的影响

这里有个特别重要的点。

GGST 的 5F Buffer 会导致：

如果你太早按第二个 P，

例如：

想延迟6F
结果提前4F按了

系统仍然会缓存。

最终出来的可能是：

延迟2F

而不是你想要的延迟6F。

所以 Leo 玩家练 delayed 5P 时通常不是数帧按，而是练节奏：

P...(停一下)...P

故意让第二个 P 晚到 Buffer 覆盖不到的地方。

因此在你给出的条件下：

Leo 5P→5P 最速是 −5F（真连防）
	​


而要得到社区常说的 1~5F Gap：

第二个P要比最速取消晚 6∼10F
	​


这就是训练模式里应该对应的放帧区间。
